# Glimmung Design System

> *The Glimmung scanned the assembled list of beings he had summoned. From a thousand worlds they had come, each with a craft to contribute.*
> — paraphrased from Philip K. Dick, *Galactic Pot-Healer*

This is the design system for **Glimmung** — an agent-run dispatcher and verify-loop orchestrator. Glimmung sits between humans-as-curators and a fleet of self-hosted agentic runners, owning the queue, the lease lifecycle, and the dashboard for issue-driven agentic software development.

The product is similar in spirit to Spacelift / Terraform Cloud or Airflow, but for orchestrating LLM-agent runs (Claude Code, Codex, etc.) against GitHub Issues, with a verify → retry-or-advance budget loop as the one cycle in the system.

## CONTENT FUNDAMENTALS

Glimmung's voice is **terse, lowercase, technical, with one note of literary affectation** (the PKD epigraph that sits in the dashboard header).

### Casing

- **Lowercase everywhere by default.** Page title is `glimmung`. Tab labels are `capacity`, `issues`, `prs`. Buttons say `sign in`, `admin`, `dispatch`, `cancel`, `keep`, `refresh`.
- **Uppercase ONLY for tiny eyebrow labels** in tables and key/value rows: `NAME`, `CAPABILITIES`, `STATE`, `LAST HEARTBEAT`, `PROJECTS`. These use `text-transform: uppercase` + `letter-spacing: 0.04–0.1em` and a muted gray (`#666`/`#888`).
- **Section headings (`h2`) are uppercase + tracked.** Body headings (`h1`) stay lowercase.
- **Sentence case for inline copy and empty states** ("No hosts registered. Sign in and use the admin panel to add one.").

### Voice and pronouns

- The dashboard speaks in **third-person system narration** — it describes what it sees, not what "you" should do. Examples: "No leases waiting." / "No active leases." / "no workflows" / "No open issues across registered repos."
- Imperatives appear sparingly, only for setup gates: "Sign in to view issues." / "Sign in and use the admin panel to add one."
- Never says "I". Almost never says "you" outside of those gate prompts.
- README + design docs use first-person plural ("we dispatch a venue", "glimmung owns the queue") — implementer voice, not user-facing voice.

### Tone

- **Engineer-to-engineer.** Assumes the reader knows what a lease, a workflow_dispatch, an SSE stream, an _etag, and a workload identity are. Never explains acronyms. Never adds marketing fluff.
- **Compressed and precise.** Tables, key/value lists, mono IDs. No paragraph-long descriptions where a 4-row table would do.
- **Mildly literary.** The PKD Galactic Pot-Healer epigraph runs along the top of the dashboard header. The product name itself is a science-fiction reference. This single grace note is the only ornament; the rest of the UI is deliberately plain.
- **Honest about state.** Connection pill says `live` / `stale` / `dead`, not "All systems operational". Hosts are `free` / `busy` / `drained`. PRs are `passed` / `in_progress` / `aborted` / `manual`. The system never euphemizes.

### Mechanical conventions

- **IDs are truncated to 8 chars + ellipsis** in tables (`01HKQX2A…`) — full IDs only in detail views or when copied.
- **Time is relative**, lowercase, no spaces around units: `12s ago`, `4m ago`, `2h ago`, `3d ago`, `never`, `just now`. Generated by a single `relTime()` helper.
- **JSON is shown raw, monospaced, dimmed.** Capability bags and metadata are stringified inline rather than pretty-printed. The reader is assumed to be able to parse `{"gpu":true,"os":"linux"}` at a glance.
- **Em-dashes (`—`) for "no value"** in tables. Never `null`, never `N/A`, never empty cells.
- **Numbers in pill counts use `font-variant-numeric: tabular-nums`** so columns align without a fixed width.

### Vocabulary (ALWAYS use these terms)

| Use | Don't use |
|---|---|
| **lease** | reservation, booking, slot |
| **host** | machine, runner, agent, node, worker |
| **workflow** | job, pipeline, action |
| **project** | repo, app, service |
| **dispatch** | trigger, kick off, start |
| **release** | finish, complete, end |
| **drain** | take offline, cordon |
| **verify-loop** | CI, validation, testing |
| **attempt** | try, run (in retry context) |
| **signal** | event, message, notification |

### Examples (from the live product)

- Page header: `glimmung` + connection pill + epigraph + auth links — all in one row.
- Empty state: "No hosts registered. Sign in and use the admin panel to add one."
- Filter hint: "— filtered to spirelens" / "— filtered to spirelens.issue-agent" (lowercase, mono dot-separator, em-dash prefix).
- Cancel confirm: `cancel` → click → `cancel?` / `keep` (two buttons, slash separator).
- Run states (pills): `passed`, `in_progress`, `aborted`, `manual`.
- Decision outputs: `ADVANCE`, `RETRY`, `ABORT_BUDGET_ATTEMPTS`, `ABORT_BUDGET_COST`, `ABORT_MALFORMED` — these decision tokens are SHOUTING-CASE constants, the only place in the UI where uppercase is used in narrative copy.

### Emoji policy

**No emoji. Anywhere.** The codebase contains zero emoji and the visual system has no slot for them. If iconography is needed, use Lucide line icons (see ICONOGRAPHY) or a unicode symbol like `—` `…` `·`. The em-dash and ellipsis ARE the iconography for empty/truncated states.

## VISUAL FOUNDATIONS

Glimmung's visual identity is **dark, dense, terminal-adjacent**. Every value below was lifted directly from `frontend/src/index.css`.

### Color

The palette is built around **near-black backgrounds**, a single-step neutral ramp from `#050507` through `#d4d4d4`, and four semantic accents (green/amber/red/blue) used ONLY in pill badges and focus borders.

| Role | Token | Hex | Use |
|---|---|---|---|
| Page background | `--bg` | `#0e0e10` | The default canvas. |
| Sidebar / deeper surface | `--bg-deep` | `#0a0a0c` | Sidebar, attempt-card body. |
| Attempt card body | `--bg-deepest` | `#050507` | Nested-most surface. |
| Card / form surface | `--surface` | `#15151a` | Panels, hover row, selected row. |
| Surface hover | `--surface-hover` | `#1a1a1f` | Tab hover, count pill bg. |
| Border (default) | `--border` | `#2a2a2e` | All separators, table row borders, input borders. |
| Border (raised) | `--border-strong` | `#3a3a3e` | Hover input border, button border. |
| Border (focus running) | `--border-running` | `#5a3a14` | Running attempt-card border. |
| Foreground (primary) | `--fg` | `#d4d4d4` | Body text. |
| Foreground (secondary) | `--fg-secondary` | `#b8b8b8` | Project rows, mono cells. |
| Foreground (muted) | `--fg-muted` | `#999` | h2 headings, secondary labels. |
| Foreground (dim) | `--fg-dim` | `#888` | Eyebrow labels, table th. |
| Foreground (dimmer) | `--fg-dimmer` | `#666` | h2 headings inverse, cell `dim`. |
| Foreground (dimmest) | `--fg-dimmest` | `#555` | Counts, quote, empty text. |
| Foreground (faintest) | `--fg-faintest` | `#444` | Workflow-empty placeholder. |

**Semantic accents** (each is a `bg / fg` pair):

| State | Background | Foreground | Used for |
|---|---|---|---|
| Success / Free / Live / Pass | `#14532d` | `#4ade80` | `pill.free`, `connection.live`, project-row selected count, primary submit button, sidebar "selected" left-border, tab "selected" underline. |
| Busy / Stale / In-progress | `#422006` | `#fb923c` | `pill.busy`, `connection.stale`, live-dot pulse, tab unread dot. |
| Drain / Dead / Aborted / Error | `#3f1c1c` | `#f87171` | `pill.drain`, `connection.dead`, error toast, danger button text. |
| Workflow accent | `#1e3a5f` | `#60a5fa` | Workflow-row selected count, workflow selected left-border. |

There is **no brand purple, teal, or pink**. There is **no gradient anywhere** in the live UI. There is **no shadow** anywhere — depth is conveyed entirely through value (background lightness) and border.

### Type

Glimmung uses **Geist Sans + Geist Mono** — Vercel's developer-tooling family. Designed for dense dashboards: tight metrics at small sizes, tabular numerals on by default, and a slashed-zero / disambiguated `1lI` mono. Loaded from Google Fonts via `@import` at the top of `colors_and_type.css`; system stacks are fallbacks.

```css
/* Body / UI — set on :root */
font-family: "Geist", ui-sans-serif, system-ui, -apple-system, "Segoe UI",
             Roboto, "Helvetica Neue", Arial, sans-serif;

/* Mono — for IDs, capability JSON, workflow names, key fields */
font-family: "Geist Mono", ui-monospace, SFMono-Regular, Menlo, monospace;
```

Substitution disclosure: the live product still uses system stacks. Geist is the design-system upgrade — switch back by editing the two `--font-*` tokens.

Base size is **14px** (`font-size: 14px` on `:root`). The scale is compressed:

| Token | rem | px (at 14px base) | Use |
|---|---|---|---|
| `--text-xs` | 0.7rem | ~10px | Workflow-row count pill (selected). |
| `--text-2xs` | 0.75rem | ~10.5px | Sidebar title, count pills, eyebrow labels, h2. |
| `--text-sm` | 0.82–0.85rem | ~11.5–12px | Table th, attempt-card body, auth links, mono table cells. |
| `--text-base` | 0.9rem | ~12.6px | Project rows, form inputs, tab labels, button text. |
| `--text-md` | 1.05rem | ~14.7px | h2 (section heading) — yes, h2 is barely larger than body. |
| `--text-lg` | 1.6rem | ~22.4px | h1 (page title — `glimmung`). |

Headings:

- `h1` — 1.6rem, weight 600, letter-spacing -0.01em, lowercase. Always says `glimmung`.
- `h2` — 1.05rem, weight 600, color `#999`, **UPPERCASE**, letter-spacing 0.05em, margin-top 28px. e.g. `HOSTS`, `PENDING QUEUE`, `ACTIVE`, `WORKFLOW INFO`.
- Sidebar title — 0.75rem, weight 600, color `#666`, **UPPERCASE**, letter-spacing 0.1em. e.g. `PROJECTS`.
- Eyebrow keys (in `.project-info`, `.attempt-card-body`) — 0.75–0.85rem, color `#888`, **UPPERCASE**, letter-spacing 0.04em.

There is **no italic body text**. The only italic in the UI is the PKD quote in the header (`font-style: italic; color: #555`) and the "no workflows" / empty-state placeholders.

### Spacing

Driven by an 8px base; common values:

- **4px** — between adjacent same-class chips, gap between elements in a tabs strip.
- **6–8px** — vertical padding inside table cells (`8px 12px`), inside count pills (`2px 8px`), inside sidebar rows (`8px 20px`).
- **12px** — gap between form rows, inside admin form `gap`.
- **16px** — content padding on admin form, `project-info`, empty state.
- **20px** — sidebar horizontal padding.
- **24–32px** — content area padding (`24px 32px`), h2 top margin (28px), header bottom margin (32px).

Sidebar width is fixed at **220px**. Content has **min-width: 0** so tables can scroll-overflow inside flexbox.

### Borders

- **All borders are 1px solid `#2a2a2e`** unless noted. There is no border thickness scale; everything is 1px.
- **Border-radius is small.** 4px on cards, inputs, buttons, panels. 6px on the run-panel. **999px (full pill)** on count badges, connection pills, status pills, the live-dot, the tab-dot.
- **Selected state uses a 2px left or bottom border accent** (sidebar row selected: 2px solid `#4ade80` on left; tab selected: 2px solid `#4ade80` on bottom). The selected border is the ONLY way the system communicates active selection in the chrome.
- Eligible-row indicator: `tr.eligible td:first-child { border-left: 2px solid #4ade80; }` — a 2px green stripe that joins to the row.

### Shadows, blur, transparency

- **No box-shadows anywhere.** Layering is achieved by stepping the background value (`#0a0a0c` < `#0e0e10` < `#15151a` < `#1a1a1f`).
- **No backdrop-filter / blur.**
- **No transparency on chrome.** Only `:hover` on transparent buttons (transparent → `#15151a`/`#1a1a1f` solid).
- **One animation:** the `pulse` keyframe (`opacity: 1 → 0.35 → 1` over 1.6s ease-in-out infinite) on the **live-dot** (orange) and **tab-dot** (orange). That's it. No fade-ins, no slide-ins, no staggered list animation, no transitions on color or size.

### Hover, press, focus states

- **Hover** on rows / tabs / buttons darkens-but-stays-dark: transparent → `#15151a` (rows) or `#1a1a1f` (tabs); border `#2a2a2e` → `#3a3a3e` (auth-link button).
- **Selected** uses the green/blue 2px accent border + the same `#15151a` background.
- **Focus** on form inputs: `outline: none; border-color: #4ade80;` (green). No focus ring.
- **There are no `:active` / pressed states defined** in the codebase — buttons go from `:hover` straight to `:disabled` (opacity 0.5, cursor not-allowed). Don't introduce shrink, scale, or color-darken on press; that's not Glimmung.
- **Disabled** is universally `opacity: 0.5; cursor: not-allowed;`.

### Layout rules

- **Always full-bleed.** The dashboard occupies `min-height: 100vh` with sidebar + content. Don't add a max-width container.
- **Sidebar is fixed width (220px), content fills the rest.** Tables overflow within content; never a horizontal scroll on the page.
- **Header is a flex row** with: `h1` (left) → connection pill → italic quote (auto margin-left, max-width 50%, right-aligned) → auth controls. The quote is the only right-aligned element in the layout.
- **No fixed/sticky elements.** The sidebar scrolls with the page. There's no toolbar overlay.

### Imagery / illustration

- **None.** Glimmung has no photography, no illustrations, no iconography in the live product. The only graphical element on the entire dashboard is the **2px green left-stripe** on selected/eligible rows and the **pulsing orange dot**. This is a deliberate aesthetic — a CRT-terminal feel by way of restraint.
- This design system adds Lucide line icons (line weight 1.5px, `currentColor`) for use in extensions / new screens. See ICONOGRAPHY.

### Animation

- **One keyframe** (`pulse`, see above), applied to two elements (live-dot, tab-dot).
- **No CSS transitions** are declared in the codebase. State changes are instant. Don't introduce easing.
- If you absolutely must animate something new: 1.6s ease-in-out infinite is the existing rhythm; match it. Otherwise, keep state changes binary.

### Card vs panel vocabulary

| Container | Background | Border | Radius | Used for |
|---|---|---|---|---|
| Page | `#0e0e10` | — | — | Body. |
| Sidebar | `#0a0a0c` | right 1px `#2a2a2e` | — | Project nav. |
| Panel (`.admin-panel`, `.project-info`) | `#15151a` | 1px `#2a2a2e` | 4px | Inline forms, key/value cards. |
| Run panel (`.run-panel`) | `#0a0a0c` | 1px `#2a2a2e` | 6px | Verify-loop run detail. |
| Attempt card (`.attempt-card`) | `#050507` | 1px `#2a2a2e` (or `#5a3a14` if running) | 4px | Per-attempt history within a run. |
| Empty state (`.empty`) | transparent | 1px **dashed** `#2a2a2e` | 4px | The only dashed border in the system. |

## ICONOGRAPHY

Glimmung's live product **ships zero icons**. The dashboard uses pure typography + colored pills + the 2px stripe + the pulse-dot. There is no icon font, no inline SVG, no PNG, no emoji.

For this design system — and for any new screens / docs / slides that extend Glimmung — we adopt **Lucide** as the official icon set, loaded from CDN.

### Library

- **Set:** [Lucide](https://lucide.dev) — 1500+ open-source line icons, 24×24 grid, **2px stroke** (Lucide ships at 2px; we use it as-is).
- **CDN:** `https://unpkg.com/lucide-static@0.469.0/font/lucide.css` (icon font) or `https://unpkg.com/lucide@0.469.0/dist/umd/lucide.js` (inline-SVG via JS API).
- **Stroke color:** always `currentColor`. Never override stroke color directly — set `color` on the parent.
- **Sizing:** 14px (in body text), 16px (in tables / pill rows), 18px (in headers / tabs). Match the surrounding text size — never larger than the line-height.
- **Color:** match the parent text token (`#d4d4d4` for body, `#888` for muted, semantic accent for state icons). Don't tint icons independently.

### When to use icons

- **Sparingly.** The live UI ships none, so any icon is a deliberate addition. Default to text labels.
- **Allowed extensions:** retry/refresh, external-link, copy-id, expand/collapse arrows, status icons in attempt cards, sort indicators in tables.
- **Disallowed:** decorative icons next to headings, brand-style emoji, illustrative scenes, multi-color icons.

### Recommended Lucide icons for Glimmung

| Use | Icon name |
|---|---|
| Refresh / re-poll | `refresh-cw` |
| External link to GitHub | `external-link` |

| Admin (lock + cog) | `shield-check` |
| Pass | `check` |
| Fail / abort | `x` |
| In-progress | `loader` (no spin — Glimmung doesn't spin) |

| Host (machine) | `server` |
| Workflow (DAG node) | `workflow` |
| Lease | `key-round` |
| Copy | `copy` |
| Sort up / down | `chevron-up` / `chevron-down` |
| Expand / collapse | `chevron-right` / `chevron-down` |

### Logo

Glimmung has **no logo image** — only the lowercase wordmark `glimmung` rendered in 1.6rem weight 600. Treat the wordmark as the logo. See `assets/wordmark.svg` for an exact-spec SVG you can drop into screens that need a self-contained mark.

### Unicode symbols (the actual "iconography" of the live UI)

These three characters carry more visual weight in Glimmung than any icon would:

| Char | Meaning | Where |
|---|---|---|
| `—` (em-dash, `&mdash;`) | "No value" / "n/a" | Every empty table cell. |
| `…` (ellipsis, `&hellip;`) | "Truncated" / "in progress" | Truncated lease IDs, "dispatching…", "refreshing…", "Connecting…". |
| `·` (middle-dot) | Separator | Use in breadcrumbs if needed (not currently in use). |
| `?` | "Unknown state" | `?` on an unresolved run state in `IssuesView` — `${state ?? "?"}` pattern. |

Use these before you reach for an icon.

### Substitution disclosure

The live Glimmung product uses **no icons at all** — Lucide is a substitute / extension chosen for this design system. If the user wants a different icon set (Heroicons, Phosphor, Tabler), update `assets/` and this section together.

## Caveats

- **Geist is a substitution.** The live product uses system stacks; this design system standardizes on Geist Sans + Geist Mono via Google Fonts. If you'd rather stay on system fonts, edit the two `--font-*` tokens in `colors_and_type.css` and remove the `@import`.
- **No real logo asset.** The wordmark SVG in `assets/` is a clean re-rendering of the lowercase `glimmung` h1 from the dashboard, not an officially-blessed mark.
- **Lucide is a substitution.** The live product ships zero icons; we picked Lucide as the extension set. Flag if a different choice is preferred.
- **No slide template was provided**, so no `slides/` folder. The visual foundations would translate well to a dark-deck system if needed — ask if you want one generated.
