postgres: rename store package + prune dead cosmos artifacts (stage 4)

Follow-up cleanup after the per-cluster cutovers (stages 2b-2-leases)
and the Cosmos infrastructure retirement (PR #590). Everything in
this PR is mechanical removal of dead code + the package rename.

Package rename:
- internal/store/cosmos/ -> internal/store/store/. The package name
  was misleading after the migration since every method delegates
  to pg.*Store fields. Only one import site (cmd/glimmung-go/main.go)
  had to update; alias becomes glimmungstore.

Dead-code deletes (-1670 / +16 net):
- internal/store/{cosmos -> store}/{query.go, query_test.go,
  live_smoke_test.go, slot_live_smoke_test.go} — query primitives
  for Cosmos containers that no longer exist; live-smoke tests
  pinned at deleted TestLiveCosmosLockLifecycle and a dead Cosmos
  account.
- internal/store/pg/*.go — 13 *MigrationSource interfaces + 13
  Migrate(ctx, source) methods. The boot-time Migrate goroutines
  in main.go disappeared in PR #590, so no caller invokes these.
- internal/store/pg/reports.go entirely — ReportsStore was only
  reachable via Migrate; no consumer ever called any other method.
- LockMigrationRow type (only used by the deleted Migrate path).
- scripts/check-cosmos-queries.mjs — guarded approved cosmos query
  primitives; primitives are gone.

Config + workflow:
- server.Settings loses CosmosEndpoint + CosmosDatabase.
  SettingsFromEnv stops reading COSMOS_ENDPOINT / COSMOS_DATABASE.
- k8s/values.yaml, k8s/issue/values.yaml drop cosmos.endpoint +
  cosmos.database. k8s/templates/deployment.yaml +
  k8s/issue/templates/deployment.yaml drop the matching env vars.
- .github/workflows/tests.yaml: live-cosmos-smoke job removed
  (TestLiveCosmosLockLifecycle was already deleted in Stage 2b);
  check-cosmos-queries job removed (script is gone).
